# Guest UX/Product Notes (2026-02-27)

Источник: договорённости с Алексеем в чате.

## Принятые решения

1. До авторизации гость видит бот как **витрину базы**:
   - описание,
   - фото,
   - расположение,
   - проверка свободных дат,
   - путь к бронированию.

2. Кнопка авторизации должна быть всегда доступна:
   - `Уже бронировал? Авторизоваться`.

3. Добавить раздел FAQ / популярные вопросы + форму обратной связи.

4. Нейминг в личном кабинете:
   - вместо `Мои брони` использовать `Моё бронирование` (или `Моя бронь`),
   - по умолчанию ориентируемся на один активный кейс для UX-простоты.

5. `Как добраться` логически увязать с инструкцией заселения (контекстно).

6. Продуктовый принцип для всей реализации:
   - **минимум хардкода**,
   - всё через конфиг/переменные/настройки,
   - архитектура сразу под масштабирование как SaaS-продукта.

## Архитектурные требования на будущее

- Multi-tenant friendly структура (данные и контент на объект/бизнес).
- Контент и тексты — через настройки, а не захардкоженные строки.
- Кнопки/секции меню гостя — конфигурируемые.
- Feature flags для включения/отключения модулей.
- Расширяемый слой брендинга (название, контакты, медиа, правила, FAQ).
