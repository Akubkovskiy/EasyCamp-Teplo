# Guest v2 — UAT чеклист (Telegram UI)

Дата: 2026-02-28
Версия: pre-UAT

## 1) Неавторизованный гость (витрина)
- [ ] `/start` открывает витрину, а не сразу запрос контакта
- [ ] Видны кнопки: О базе / Домики и фото / Проверить даты и забронировать / FAQ / Где мы находимся / Связь / Авторизоваться
- [ ] `О базе` открывается корректно
- [ ] `Домики и фото` открывается корректно (MVP текст)
- [ ] `FAQ` открывается корректно
- [ ] `Где мы находимся` открывает текст + ссылку на карты

## 2) Авторизация
- [ ] Нажатие `Авторизоваться` просит поделиться телефоном
- [ ] Свой контакт → успешная авторизация при совпадении с бронью
- [ ] Чужой контакт → отказ
- [ ] При отсутствии брони → понятное сообщение

## 3) Авторизованный кабинет
- [ ] Показываются приоритетные кнопки: Инструкция / Связаться / Как добраться
- [ ] `Моя бронь` показывает карточку с датами/суммами
- [ ] `Оплата` показывает сумму остатка
- [ ] `Wi‑Fi` открывается
- [ ] `Правила` открываются

## 4) Оплата и чек
- [ ] `Оплата -> Отправить чек оплаты`
- [ ] Фото чека отправляется админу
- [ ] Админ может approve/reject из кнопок
- [ ] При approve: гостю приходит подтверждение, бронь становится PAID
- [ ] При reject: гостю приходит уведомление об отклонении

## 5) Инструкция по времени
- [ ] Если до заезда >24ч — инструкция закрыта с понятным текстом
- [ ] Если <=24ч — инструкция доступна

## 6) FAQ / обратная связь
- [ ] `Задать свой вопрос` предлагает выбрать категорию
- [ ] После выбора категории сообщение уходит админам
- [ ] В сообщении админам видна категория

## 7) Партнёры
- [ ] Раздел `Партнёры` доступен (если feature flag включён)
- [ ] Показан v1-каталог (текст из настроек или fallback)

## 8) Feature flags (опционально, техпроверка)
- [ ] Отключение `GUEST_FEATURE_FAQ=false` скрывает FAQ из витрины
- [ ] Отключение `GUEST_FEATURE_PARTNERS=false` скрывает Партнёров из auth-меню
- [ ] Отключение `GUEST_FEATURE_SHOWCASE_HOUSES=false` скрывает Домики и фото

## 9) Self-service бронирование (Phase G10.1)

- [ ] Unauth поток: `/start` → витрина → «📅 Проверить даты» → выбор дат → у любой карточки кнопка «✅ Забронировать»
- [ ] Нажатие → бот просит share contact (reply-keyboard «📱 Поделиться моим телефоном»)
- [ ] После share contact: бот предлагает выбрать кол-во гостей (inline-кнопки 1..capacity)
- [ ] После выбора кол-ва: карточка подтверждения с домом, датами, ценой, ФИО, телефоном
- [ ] «✅ Подтвердить» → бронь создана со статусом `NEW`, гостю «Заявка отправлена»
- [ ] Админ получает уведомление с кнопками «✅ Подтвердить» / «❌ Отклонить»
- [ ] Admin confirm → гостю приходит сообщение с кнопкой «💳 Оплата», статус `CONFIRMED`
- [ ] Admin reject → гостю приходит уведомление, статус `CANCELLED`, даты освобождены
- [ ] Auth поток (уже логинились ранее): «✅ Забронировать» сразу к выбору гостей, контакт не спрашивается
- [ ] Попытка забронировать без выбора дат → бот вежливо просит сначала выбрать даты
- [ ] Гонка: даты заняты между шагом «карточка» и «подтвердить» → бот говорит «эти даты только что заняли»
- [ ] Cancel в потоке (кнопка «❌ Отменить») → бот закрывает поток без побочных эффектов

## 10) Self-service отмена брони (Phase G10.2)

- [ ] В карточке `Моя бронь` есть кнопка «🚫 Отменить бронь»
- [ ] При <= `guest_cancel_window_days` (default 7) до заезда → экран «свяжитесь с админом» с кнопкой `📞 Связаться с нами`
- [ ] При > `guest_cancel_window_days` → подтверждение «✅ Да, отменить» / «🔙 Нет, оставить»
- [ ] После confirm → статус `CANCELLED`, даты в Avito разблокированы, sheets обновлены
- [ ] Гостю приходит подтверждение «Бронь #N отменена»
- [ ] Админ получает уведомление «Гость отменил бронь #N»
- [ ] Попытка отменить чужую бронь (несовпадающий телефон) → отказ «Это не ваша бронь»
- [ ] Replay старой кнопки после прошедшего окна → отказ «Окно отмены закрыто»

## 11) Конфигурируемые окна (Phase G10.3)

- [ ] Установить `GlobalSetting(guest_instruction_open_hours='48')` → инструкция открывается за 48ч
- [ ] Сообщение в карточке инструкции отражает фактическое окно («открывается за 48 ч»)
- [ ] Установить `GlobalSetting(guest_cancel_window_days='14')` → cancel-flow использует 14 дней
- [ ] Удалить ключи / поставить пустое значение → используются дефолты (24ч / 7дн)
- [ ] Поставить нечисловое значение → fallback на дефолт + warn в логах
